Configuring this repository

Test settings live in your .viancfg files - this page does not change them. You create and edit VI Analyzer test configurations in LabVIEW (Tools › VI Analyzer › Analyze VIs…, choose the tests and exclusions, then Save Configuration) and commit the resulting .viancfg files to this repository. This page only tells the automated pipeline:
  • which VIs get analyzed at all;
  • which committed .viancfg applies to a specific VI or folder;
  • which .viancfg (if any) is used by default when nothing else matches.
VI Analyzer is not currently an enabled activity for this repository. These mappings are saved, but nothing runs until VI Analyzer is turned on in Configure Workers › Pipeline.

Default configuration

Applied to every VI that isn't matched by a rule below. The built-in full suite runs all default VI Analyzer tests; pick a committed .viancfg to use its tests instead, or choose don't test by default so only files matched by a rule are analyzed.

Per-folder & per-file configurations

Assign a specific .viancfg to a subset of your code — choose individual VIs and/or whole folders. Files matched by a rule are analyzed with that configuration instead of the default. Choose Exclude to skip the selected files entirely (some VIs not being tested is fine). When a file matches more than one rule, the first rule listed wins.

Apply

A static page can't write to your repo, so run this command to apply the change. It runs the Reconfigure workflow, which updates only the VI Analyzer settings in .github/labview-ci.yml (every other setting is preserved) and opens a pull request to review. It uses GitHub's own auth — no tokens touch the browser.
